### 좋았던 점, 좋지 않았던 점

<details open><summary>..닫기</summary>

✨ 장점:

* 책의 설명이 아주 상세하고 친절하기 때문에 어렵지 않게 끝까지 읽을 수 있습니다.

* TDD 뿐만 아니라, 애자일 개발 프로세스, 프로비저닝, 지속적인 통합 등 상당히 풍부한 내용을 담고 있습니다. 

* 테스트를 아주 촘촘히 작성하기 때문에, 웹 프레임워크의 내부 동작에 대해서도 자세히 알 수 있습니다. 

<br/>

✨ 단점:

* 책에서 사용한 Mozilla의 Persona 서비스가 중단되어 15장, 16장의 2 factor 인증 모킹 부분은 영문판 2판을 참조해야 했습니다. 

* 실습 코드에서 편집 실수가 몇 개 발견되지만, 실습하는데 크게 무리가 없는 정도입니다.
